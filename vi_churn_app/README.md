# 📡 Vi Telecom Customer Churn Intelligence Dashboard

> A production-grade Streamlit ML dashboard for predicting and interpreting customer churn for Vi (Vodafone Idea) Telecom India, powered by XGBoost, Random Forest, Decision Tree, and Claude AI-generated insights.

---

## 🚀 Features

| Tab | Contents |
|-----|----------|
| **Overview** | KPI cards (Churn Rate, Avg Charges, Avg Tenure, High-Risk Segment) + Top 5 States by Churn |
| **Customer EDA** | Stacked bar (Plan×Gender), Violin (Monthly Charges), Box (Tenure), Correlation Heatmap |
| **Churn Models** | Metrics table, Confusion Matrices, Overlaid ROC Curves |
| **Retention Insights** | Feature Importance (3 models), Permutation Importance, Top 5 churn drivers + business actions |

**AI-powered:** Every chart has a 2-line insight generated by Claude AI (Anthropic), regenerated automatically when sidebar filters change.

## ⚡ Quick Start

### 1. Clone / unzip the project

```bash
git clone <your-repo-url>
cd vi_churn_app
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Pre-train models (optional but recommended for faster dashboard load)

```bash
python train.py
```

This runs the full ML pipeline (Steps 1–9) and saves models to `models/`.  
If skipped, the dashboard trains on first load (takes ~30 seconds).

### 4. Set your Anthropic API key

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

Or add it to Streamlit secrets (see Deployment section below).

### 5. Run the dashboard

```bash
streamlit run app.py
```

## 🛰️ Streamlit Cloud Deployment

1. Push this folder to a GitHub repository.
2. Go to [share.streamlit.io](https://share.streamlit.io) → New App → select your repo → `app.py`.
3. Under **Secrets**, add:
   ```toml
   ANTHROPIC_API_KEY = "sk-ant-..."
   ```
4. Click **Deploy**.

> **Note:** The `data/` and `models/` folders must be committed to GitHub. The pre-trained `.joblib` files (~5–15 MB each) make deployment faster.

## 📊 ML Pipeline Steps

| Step | Description |
|------|-------------|
| 1 | Import packages |
| 2 | Load & basic checks (snake_case, coerce numerics, dedup, EDA) |
| 3 | Handle missing values (median/mean/mode) + imputation flags |
| 4 | Encode categoricals (OHE, ordinal, frequency) + StandardScaler |
| 5 | Feature engineering (usage_ratio, complaint_rate, tenure_bucket, payment_risk_score) |
| 6 | Train/test split 80:20, stratified |
| 7 | Train Decision Tree, Random Forest, XGBoost |
| 8 | Evaluate (Accuracy, Precision, Recall, F1, ROC AUC, PR AUC, confusion matrices, curves) |
| 9 | Explainability (feature importance, permutation importance) |
